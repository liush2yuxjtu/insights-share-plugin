# 1. Technical Findings & Patterns

> author: m1 · confidence: 0.85


### 1.1 Sprint Contract Pattern (Universal)

Every project uses a **consistent 7-section Sprint Contract** format:

```
1. Objective
2. Scope (In Scope / Out of Scope)
3. Files Likely Affected
4. Done Criteria (independently verifiable)
5. Verification Plan (with actual commands)
6. Risks (with likelihood, impact, mitigation)
7. [Optional] Review verdict (APPROVED / REVISE)
```

**Key insight**: Done Criteria MUST be independently verifiable with specific pass conditions.
